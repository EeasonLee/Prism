# 前端购物车合并功能集成指南

## 概述

后端已实现登录/注册时自动合并游客购物车的功能。本文档说明前端需要做的调整。

---

## API 变更说明

### 1. 登录接口 `POST /api/auth/login`

#### 新增请求参数（可选）

| 参数             | 类型    | 必填 | 说明             | 格式要求                           |
| ---------------- | ------- | ---- | ---------------- | ---------------------------------- |
| `guestSsoUserId` | string  | 否   | 游客 SSO 用户 ID | 必须符合 `^guest-[a-f0-9-]+$` 格式 |
| `storeId`        | integer | 否   | Magento 店铺 ID  | 不传则使用后端默认配置             |

#### 新增响应字段

| 字段              | 类型   | 说明           | 可能值                           |
| ----------------- | ------ | -------------- | -------------------------------- |
| `cartMergeStatus` | string | 购物车合并状态 | `success` / `failed` / `skipped` |

#### 请求示例

```typescript
const response = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'password123',
    guestSsoUserId: 'guest-6df16ccb-43e4-45ef-b111-ec2fb501f99d', // 可选
    storeId: 1, // 可选
  }),
});
```

#### 响应示例

```json
{
  "user": {
    "id": "02cd9c36-c0a3-4890-9087-a8b70446fa90",
    "email": "user@example.com",
    "username": "user",
    "role": "customer",
    ...
  },
  "tokens": {
    "accessToken": "eyJhbGciOiJIUzI1NiJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiJ9..."
  },
  "cartMergeStatus": "success"
}
```

#### 新增响应字段

与登录接口相同：`cartMergeStatus`

## 前端集成步骤

### 步骤 1: 游客登录时保存 `guest_id`

当用户以游客身份访问时，调用游客登录接口并保存返回的 `guest_id`：

```typescript
// 游客登录
const guestResponse = await fetch('/api/auth/guest', {
  method: 'POST',
});

const { guest_id, tokens } = await guestResponse.json();

// 保存游客 ID 和 token
localStorage.setItem('guest_sso_user_id', guest_id);
localStorage.setItem('access_token', tokens.accessToken);
localStorage.setItem('refresh_token', tokens.refreshToken);
```

**重要**: `guest_id` 的格式为 `guest-{uuid}`，例如 `guest-6df16ccb-43e4-45ef-b111-ec2fb501f99d`

---

### 步骤 2: 登录时传入 `guestSsoUserId`

用户登录时，检查本地是否存在游客 ID，如果有就传给后端：

```typescript
const guestSsoUserId = localStorage.getItem('guest_sso_user_id');

const loginResponse = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email,
    password,
    guestSsoUserId: guestSsoUserId || undefined, // 没有就不传
    storeId: 1, // 可选，不传则使用默认
  }),
});

const loginData = await loginResponse.json();
```

---

### 步骤 3: 注册时同样传入 `guestSsoUserId`

用户注册时的处理与登录相同：

```typescript
const guestSsoUserId = localStorage.getItem('guest_sso_user_id');

const registerResponse = await fetch('/api/auth/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email,
    password,
    first_name,
    last_name,
    guestSsoUserId: guestSsoUserId || undefined,
    storeId: 1,
  }),
});

const registerData = await registerResponse.json();
```

---

### 步骤 4: 处理 `cartMergeStatus` 响应

根据返回的 `cartMergeStatus` 做相应处理：

```typescript
const data = await response.json();

if (data.user && data.tokens) {
  // 保存用户 token
  localStorage.setItem('access_token', data.tokens.accessToken);
  localStorage.setItem('refresh_token', data.tokens.refreshToken);

  // 根据购物车合并状态处理
  switch (data.cartMergeStatus) {
    case 'success':
      // 合并成功，清除游客 ID
      localStorage.removeItem('guest_sso_user_id');
      console.log('购物车合并成功');
      // 可选：刷新购物车显示
      // refreshCart();
      break;

    case 'failed':
      // 合并失败，但登录/注册已成功
      console.warn('购物车合并失败，但登录成功');
      localStorage.removeItem('guest_sso_user_id'); // 仍然清除，避免重复尝试
      // 可选：提示用户刷新购物车
      // showNotification('购物车同步失败，请刷新页面');
      break;

    case 'skipped':
      // 没有游客购物车需要合并，无需处理
      console.log('未提供游客购物车，跳过合并');
      break;
  }
}
```

---

## cartMergeStatus 状态说明

| 状态      | 说明                              | 前端处理建议                                 |
| --------- | --------------------------------- | -------------------------------------------- |
| `success` | 购物车合并成功                    | 清除 `guest_sso_user_id`，可选刷新购物车显示 |
| `failed`  | 购物车合并失败（但登录/注册成功） | 清除 `guest_sso_user_id`，可选提示用户       |
| `skipped` | 未提供 `guestSsoUserId`，跳过合并 | 无需处理                                     |

## 完整示例代码

### React 示例

```typescript
import { useState } from 'react';

function LoginForm() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');

  const handleLogin = async (e: React.FormEvent) => {
    e.preventDefault();

    // 获取游客 ID（如果有）
    const guestSsoUserId = localStorage.getItem('guest_sso_user_id');

    try {
      const response = await fetch('/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          email,
          password,
          guestSsoUserId: guestSsoUserId || undefined,
          storeId: 1,
        }),
      });

      const data = await response.json();

      if (data.user && data.tokens) {
        // 保存 token
        localStorage.setItem('access_token', data.tokens.accessToken);
        localStorage.setItem('refresh_token', data.tokens.refreshToken);

        // 处理购物车合并状态
        if (data.cartMergeStatus === 'success') {
          localStorage.removeItem('guest_sso_user_id');
          console.log('购物车合并成功');
        } else if (data.cartMergeStatus === 'failed') {
          localStorage.removeItem('guest_sso_user_id');
          console.warn('购物车合并失败');
        }

        // 跳转到首页或其他页面
        window.location.href = '/';
      }
    } catch (error) {
      console.error('登录失败:', error);
    }
  };

  return (
    <form onSubmit={handleLogin}>
      <input
        type="email"
        value={email}
        onChange={e => setEmail(e.target.value)}
        placeholder="邮箱"
      />
      <input
        type="password"
        value={password}
        onChange={e => setPassword(e.target.value)}
        placeholder="密码"
      />
      <button type="submit">登录</button>
    </form>
  );
}
```

## 注意事项

### 1. 格式验证

- `guestSsoUserId` 必须符合正则表达式 `^guest-[a-f0-9-]+$`
- 后端会验证格式，不符合会返回 400 错误

### 2. 向后兼容

- 所有新增参数都是**可选的**
- 不传 `guestSsoUserId` 和 `storeId` 不会影响任何现有功能
- `user` 和 `tokens` 的结构完全不变

### 3. 错误处理

- 购物车合并失败**不会影响**登录/注册流程
- 即使 `cartMergeStatus` 为 `failed`，用户仍然成功登录/注册
- 建议无论成功还是失败，都清除 `guest_sso_user_id`，避免反复用过期的游客 ID 尝试合并

### 4. 清理策略

建议在以下情况清除 `guest_sso_user_id`：

- 登录/注册成功后（无论合并成功与否）
- 用户主动登出时
- Token 过期时

### 5. 测试建议

测试场景：

1. **游客添加商品 → 登录 → 验证购物车合并**

   - 以游客身份添加商品到购物车
   - 登录时传入 `guestSsoUserId`
   - 验证登录后购物车包含游客添加的商品

2. **游客添加商品 → 注册 → 验证购物车合并**

   - 以游客身份添加商品到购物车
   - 注册时传入 `guestSsoUserId`
   - 验证注册后购物车包含游客添加的商品

3. **直接登录（无游客购物车）**

   - 不传 `guestSsoUserId`
   - 验证 `cartMergeStatus` 为 `skipped`

4. **合并失败场景**
   - 传入无效的 `guestSsoUserId`
   - 验证登录仍然成功，`cartMergeStatus` 为 `failed`
